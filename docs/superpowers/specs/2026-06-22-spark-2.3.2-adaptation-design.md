# Spark 2.3.2 引擎适配设计

> **⚠ 已废弃 (2026-06-23)**：Spark 已升级到 3.5.8，Python 降级到 3.11，主进程原生运行 PySpark。子进程桥接方案不再需要。详见 `pyproject.toml` 中的 `pyspark>=3.5.8,<4.0`。

**日期**: 2026-06-22
**目标**: ~~让 BiliAnalysis 的 Spark 分析引擎在 Spark 2.3.2 集群上完整运行，与 PandasEngine 对等~~

## 背景

项目已有 `SparkEngine`（`src/bilianalysis/engine/spark_engine.py`，599 行），实现了 4/5 分析步骤（除 `model_comparison`）。但存在两个核心问题：

1. **Python 版本冲突**：主项目使用 Python 3.13，Spark 2.3.2 最高只支持 Python 3.7
2. **Spark 3.x 专属配置**：代码使用了 `spark.sql.adaptive.*` 等 2.3.2 中不存在的配置

`model_comparison` 保持 `NotImplementedError`，仅 Pandas 引擎支持——因为 Spark 2.3.2 MLlib 不提供 sklearn 级别的多模型交叉验证能力。

## 核心方案：子进程隔离

主进程（Python 3.13）通过子进程调用 Python 3.7 环境中的 PySpark 2.3.2，使用 JSON/Parquet 作为数据交换格式。

### 为什么选择子进程而非 Docker 或集群升级

| 方案 | 问题 |
|------|------|
| Docker 容器 | 增加运维复杂度，Windows 上 Docker Desktop 资源开销大，文件挂载路径兼容性差 |
| 升级到 Spark 3.5+ | 涉及集群运维决策，超出项目范围 |
| **子进程隔离** | 代码量最小（~250 行），JSON/Parquet 天然适配，两个环境互不污染 |

## 文件结构

```
src/bilianalysis/engine/
├── __init__.py                  # create_engine() factory（修改）
├── base.py                      # AnalysisEngine ABC + 报告模型（不变）
├── pandas_engine.py             # PandasEngine（不变）
└── spark/                       # NEW 子包
    ├── __init__.py              # 导出 SparkBridge, SparkStepError, SparkRuntimeError
    ├── bridge.py                # SparkBridge — 运行在 Python 3.13，实现 AnalysisEngine
    ├── engine.py                # SparkEngine — 从 spark_engine.py 迁入，适配 Spark 2.3.2
    ├── entry.py                 # CLI 入口，子进程启动点
    └── pyproject.toml           # Spark 子环境依赖（Python 3.7）
```

### 职责边界

| 文件 | 运行环境 | 依赖 PySpark | 职责 |
|------|---------|-------------|------|
| `bridge.py` | Python 3.13 | ❌ | 实现 `AnalysisEngine`，每个方法内 `subprocess.run` 调 `entry.py` |
| `engine.py` | Python 3.7 | ✅ | 纯 Spark 逻辑，与现有 `SparkEngine` 一致，仅做 2.3.2 兼容适配 |
| `entry.py` | Python 3.7 | ✅ | stdin 读 step 名 + 配置 → 实例化 `SparkEngine` → 调对应方法 → stdout 输出 JSON |

`bridge.py` 不 `import pyspark`，确保 Python 3.13 主进程不会被 PySpark 2.3.2 依赖污染。

## 子进程协议

### 输入（bridge → entry，stdin JSON）

```json
{
  "step": "statistics",
  "raw_dir": "/data/raw",
  "processed_dir": "/data/processed",
  "reports_dir": "/data/reports",
  "spark_master": "local[*]"
}
```

### 输出（entry → bridge，stdout JSON）

成功：
```json
{
  "ok": true,
  "data": { ... }
}
```

失败：
```json
{
  "ok": false,
  "error": "Traceback ..."
}
```

`data` 内容为对应 Pydantic 报告模型的 `.model_dump()` 结果。

## 组件设计

### 1. SparkBridge (`bridge.py`)

```python
class SparkBridge(AnalysisEngine):
    def __init__(self, data_config, spark_config):
        self._data = data_config
        self._py37 = spark_config.python       # Python 3.7 venv 路径
        self._master = spark_config.master     # local[*] 或 spark://...
        self._timeout = spark_config.timeout   # 秒

    def _run_step(self, step: str) -> dict:
        """调起 py37 子进程，返回 parsed JSON data"""
        req = { ... step, dirs, master ... }
        proc = subprocess.run(
            [self._py37, "-m", "bilianalysis.engine.spark.entry"],
            input=json.dumps(req),
            capture_output=True, text=True, timeout=self._timeout,
        )
        result = json.loads(proc.stdout)
        if not result["ok"]:
            raise SparkStepError(step, result["error"])
        return result["data"]

    async def clean_data(self) -> CleanReport:
        loop = asyncio.get_event_loop()
        result = await loop.run_in_executor(None, self._run_step, "clean_data")
        return CleanReport(**result)

    def statistics(self) -> StatReport:
        return StatReport(**self._run_step("statistics"))

    def clustering(self) -> ClusterReport:
        # 嵌套 Pydantic 模型，model_validate 自动处理
        return ClusterReport(**self._run_step("clustering"))

    def prediction(self) -> PredictionReport:
        return PredictionReport(**self._run_step("prediction"))

    def model_comparison(self):
        raise NotImplementedError(...)
```

- `clean_data` 的 `async def` 通过 `run_in_executor` 包装同步 subprocess 调用
- 其他方法是同步的，直接调用
- 所有 Pydantic 构造使用 `**dict` 展开，纯 JSON 类型无需额外转换

### 2. entry.py

```python
import sys, json, asyncio
from bilianalysis.engine.spark.engine import SparkEngine
from bilianalysis.config.model import DataSection

def main():
    req = json.loads(sys.stdin.read())
    data_config = DataSection(
        raw_dir=req["raw_dir"],
        processed_dir=req["processed_dir"],
        reports_dir=req["reports_dir"],
    )
    engine = SparkEngine(data_config, spark_master=req.get("spark_master", "local[*]"))

    step = req["step"]
    method = getattr(engine, step)

    if asyncio.iscoroutinefunction(method):
        result = asyncio.run(method())
    else:
        result = method()

    print(json.dumps({"ok": True, "data": result.model_dump()}, default=str))

if __name__ == "__main__":
    try:
        main()
    except Exception as e:
        import traceback
        print(json.dumps({"ok": False, "error": traceback.format_exc()}))
        sys.exit(1)
```

### 3. Spark 2.3.2 兼容性修改 (`engine.py`)

| 问题 | 变更 | 行数 |
|------|------|------|
| `spark.sql.adaptive.*` 不存在 | 删除两行 `.config()` | -2 |
| `config.start_time` / `config.end_time` schema 不一致 | 改为 `config.stime` / `config.etime`，显式 cast 类型 | ~8 |
| `master` 硬编码 | 改为构造函数参数 `spark_master` | ~5 |
| 4 次独立 `monotonically_increasing_id()` 导致 row_id 不一致 | 在一个 DataFrame 上先生成 row_id，再派生 4 张表 | ~15 |
| HDFS 可选 | 保持现有可选逻辑，默认不启用 | 0 |

### 4. 错误处理

| 场景 | 异常类 | 处理 |
|------|--------|------|
| Python 3.7 不存在 | `SparkRuntimeError("python not found: {path}")` | subprocess 抛 `FileNotFoundError` |
| 超时 | `SparkStepError("{step} timed out")` | subprocess 抛 `TimeoutExpired` |
| Spark 进程崩溃 | `SparkStepError("{step} failed:\n{stderr}")` | returncode != 0 |
| 业务异常 | `SparkStepError("{step} failed:\n{error}")` | JSON `ok: false` |
| stdout 非 JSON | `SparkStepError("{step} failed: {stdout_preview}")` | JSONDecodeError |

异常类定义在 `engine/spark/__init__.py`：

```python
class SparkRuntimeError(RuntimeError):
    """Spark 子进程运行环境错误（如解释器路径错误）。"""

class SparkStepError(RuntimeError):
    """Spark 步骤执行失败。"""
    def __init__(self, step: str, detail: str):
        super().__init__(f"Spark step '{step}' failed:\n{detail}")
```

### 6. 工厂适配

`src/bilianalysis/engine/__init__.py`：

```python
def create_engine(config: AppConfig) -> AnalysisEngine:
    if config.analysis.engine == "spark":
        from bilianalysis.engine.spark.bridge import SparkBridge
        return SparkBridge(config.data, config.spark)
    return PandasEngine(config.data)
```

SparkEngine 不再从 `__init__.py` 直接导入——仅 `entry.py` 在 Python 3.7 子进程中 import。
