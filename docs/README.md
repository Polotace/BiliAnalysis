# Bilibili“每周必看”栏目数据分析平台项目方案

## 一、项目背景

随着视频平台的快速发展，Bilibili 已成为国内最具影响力的视频社区之一。其中，“每周必看”栏目汇聚了每周最受用户关注和欢迎的视频内容，具有较高的数据分析价值。

通过对“每周必看”栏目的历史数据进行采集、清洗、存储、统计分析及数据挖掘，可以发现视频内容传播规律、热门分区趋势、UP主创作特征以及用户偏好变化，为内容推荐、创作者运营和平台发展提供数据支持。

本项目基于 Python 生态构建完整的数据分析平台，结合 Pandas、PySpark、机器学习算法以及现代 Web 可视化技术，实现从数据采集到分析展示的完整流程。

------

# 二、项目目标

构建一个面向 Bilibili“每周必看”栏目的数据分析平台，实现：

1. 视频数据采集与管理
2. 数据清洗与预处理
3. 视频统计分析
4. 热门分区分析
5. UP主分析
6. 时间趋势分析
7. 聚类分析
8. 播放量预测分析
9. 数据可视化展示
10. Pandas 与 Spark 双分析引擎切换

------

| favorite     | 收藏数   |

- 总点赞量

## 5.6 时间趋势分析

按周统计：

- 平均播放量变化
- 平均点赞量变化
- 平均互动率变化

分析目标：

观察内容热度演变趋势。

------

## 5.7 聚类分析模块

### KMeans聚类

特征：

- view
- like
- coin
- favorite

聚类结果：

#### 类别1

爆款视频

特点：

高播放、高互动

#### 类别2

普通热门视频

特点：

播放量较高

互动中等

#### 类别3

潜力视频

特点：

播放量一般

互动率较高

分析意义：

识别不同传播类型的视频内容。

------

## 5.8 播放量预测模块

采用：

### 线性回归模型

预测：

- 播放量趋势
- 点赞量趋势

分析价值：

预测未来内容热度变化。

# 六、分析引擎设计

## 6.1 设计目标

为保证系统扩展性，实现统一分析接口。

------

## 6.2 抽象引擎

定义：

AnalysisEngine

核心功能：

- clean_data()
- statistics()
- clustering()
- prediction()

------

## 6.3 Pandas分析引擎

实现：

PandasEngine

特点：

- 开发简单
- 资源消耗低
- 适合中小规模数据

------

## 6.4 Spark分析引擎

实现：

SparkEngine

特点：

- 分布式计算
- 海量数据处理
- 支持MLlib

------

## 6.5 引擎切换机制

配置文件：

```yaml
analysis:
  engine: spark
```

或

```yaml
analysis:
  engine: pandas
```

系统启动时自动加载对应分析引擎。
